## Sobre O Projeto

[![Captura de Tela do Produto][product-screenshot]](https://example.com)

Existem muitos ótimos templates de README disponíveis no GitHub; no entanto, não encontrei um que realmente atendesse às minhas necessidades, então criei este aprimorado. Quero criar um template de README tão incrível que seja o último que você vai precisar -- acho que é este.

Aqui está o porquê:

- Seu tempo deve ser focado em criar algo incrível. Um projeto que resolve um problema e ajuda outras pessoas
- Você não deveria fazer as mesmas tarefas repetidas vezes, como criar um README do zero
- Você deve implementar os princípios DRY para o resto da sua vida :smile:

Claro, nenhum template servirá para todos os projetos, pois suas necessidades podem ser diferentes. Então eu estarei adicionando mais no futuro próximo. Você também pode sugerir alterações fazendo um fork deste repositório e criando um pull request ou abrindo uma issue. Obrigado a todas as pessoas que contribuíram para expandir este template!

Use o `BLANK_README.md` para começar.

## Começando

Este é um exemplo de como você pode dar instruções sobre como configurar o seu projeto localmente.
Para ter uma cópia local rodando e funcionando siga estes passos simples de exemplo.

### Pré-requisitos

Este é um exemplo de como listar as coisas que você precisa para usar o software e como instalá-las.

- npm
  ```sh
  npm install npm@latest -g
  ```

### Instalação

_Abaixo está um exemplo de como você pode instruir o seu público na instalação e configuração do seu aplicativo. Este template não depende de dependências externas ou serviços._

1. Obtenha uma API Key gratuita em [https://example.com](https://example.com)
2. Clone o repositório
   ```sh
   git clone https://github.com/alissonpef/Flask-PDF-Watermarker.git
   ```
3. Instale os pacotes NPM
   ```sh
   npm install
   ```
4. Insira a sua API no arquivo `config.js`
   ```js
   const API_KEY = 'INSIRA SUA API';
   ```
5. Mude a URL do remote git para evitar pushs acidentais para o projeto base
   ```sh
   git remote set-url origin github_username/repo_name
   git remote -v # confirme as mudanças
   ```

## Contribuindo

As contribuições são o que tornam a comunidade open source um lugar tão incrível para aprender, inspirar e criar. Qualquer contribuição que você fizer será **muito apreciada**.

Se você tiver alguma sugestão que tornaria isso melhor, por favor faça o fork do repositório e crie um pull request. Você também pode simplesmente abrir uma issue com a tag "enhancement".
Não se esqueça de dar uma estrela ao projeto! Obrigado novamente!

1. Faça o Fork do Projeto
2. Crie a sua Branch de Funcionalidade (`git checkout -b feature/FuncionalidadeIncrivel`)
3. Commit suas Mudanças (`git commit -m 'Adicione alguma FuncionalidadeIncrivel'`)
4. Faça o Push para a Branch (`git push origin feature/FuncionalidadeIncrivel`)
5. Abra um Pull Request
